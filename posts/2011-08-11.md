## pthread

一人の学生さんが OpenCV と OpenGL を組み合わせて何かしようと考えているらしいのですが，どうも動きがぎこちなく見えます．その学生さんは MacBook Pro を使っているのですが，1 フレームキャプチャするごとにデフォルトでは 60-70ms ブロックしてしまいます．15fps ですね．それと OpenGL のダブルバッファリングのスワップバッファのタイミングが合わないために，表示のフレームレートが一定せず，ぎこちなく見えるのではないかと思います．CV_CAP_PROP_FPS に 30.0 を設定しても，やっぱり微妙にガタガタします．
ということで，とりあえず自分でもテストプログラムを書いてみました．

## この辺みんなどうやってんのかなーと調べてみてもよくわからず（どうすべきなのかご存知でしたら教えてください），仕方なくキャプチャを別スレッドで回して，キャプチャと表示を非同期に行うということを試してみました．

まず，カメラからのキャプチャを無限ループにします．cvGrabFrame() がすぐに帰ってくるとループがぶんぶん回ってしまってまずい気がしますが，必ず１フレーム分の時間がかかっているみたいなので，何も考えずに回します．また，キャプチャした画像は配列 texture に保存しておきます．このタイミングがクリティカルセクションになるので，mutex でロックしておきます．

<pre>...
 
<em>// テクスチャ</em>
<em>static GLenum format;</em>
<em>static GLsizei width, height;</em>
<em>static GLubyte texture[TEXHEIGHT * TEXWIDTH * 4];</em>
 
<em>// スレッド</em>
<em>#include &lt;pthread.h&gt;</em>
<em>static pthread_t thread;</em>
<em>static pthread_mutex_t mutex;</em>
 

## /*

* テクスチャ作成
*/
static void <em>*</em>getTexture(<em>void *arg</em>)
{
<em>CvCapture *capture = reinterpret_cast&lt;CvCapture *&gt;(arg);</em>
  
<em>for (;;) {</em>
if (cvGrabFrame(capture)) {
      

## // キャプチャ映像から画像を切り出す

IplImage *image = cvRetrieveFrame(capture);
      

## if (image) {

// 切り出した画像の書式を保存する
<em>width = image-&gt;width;</em>
<em>height = image-&gt;height;</em>
if (image-&gt;nChannels == 3)
format = GL_BGR;
else if (image-&gt;nChannels == 4)
format = GL_BGRA;
else
format = GL_LUMINANCE;
        

## // 切り出した画像を保存する

<em>GLsizei size = image-&gt;width * image-&gt;nChannels;</em>
<em>pthread_mutex_lock(&amp;mutex);</em>
for (int y = 0; y &lt; image-&gt;height; ++y)
<em>memcpy(texture + size * y, image-&gt;imageData + image-&gt;widthStep * y, size);</em>
<em>pthread_mutex_unlock(&amp;mutex);</em>
}
}
  
<em>pthread_testcancel();</em>
<em>}</em>
  
<em>return 0;</em>
}
 
```


## これを別スレッドで走らせます．mutex も作っておきます．また，プログラム終了時の処理も用意しておきます．


<pre>...
 

## /*

* プログラム終了時の処理
*/
static void releaseCapture(void)
{
<em>// スレッドを停止する</em>
<em>pthread_cancel(thread);</em>
  
<em>// スレッドの停止を待つ</em>
<em>pthread_join(thread, 0);</em>
  
<em>// ミューテックスを破棄する</em>
<em>pthread_mutex_destroy(&amp;mutex);</em>
  

## // image の release

cvReleaseCapture(&amp;capture);
}
 

## /*

* OpenCV の初期化
*/
static void cvInit(void)
{
// カメラを初期化する
capture = cvCreateCameraCapture(CV_CAP_ANY);
if (capture == 0) {
std::cerr &lt;&lt; "cannot capture image" &lt;&lt; std::endl;
exit(1);
}
cvSetCaptureProperty(capture, CV_CAP_PROP_FRAME_WIDTH, WIDTH);
cvSetCaptureProperty(capture, CV_CAP_PROP_FRAME_HEIGHT, HEIGHT);
cvSetCaptureProperty(capture, CV_CAP_PROP_FPS, 30.0);
  
<em>// スレッドを生成する</em>
<em>pthread_mutex_init(&amp;mutex, 0);</em>
<em>pthread_create(&amp;thread, 0, getTexture, capture);</em>
  

## // プログラム終了時に capture を release する

atexit(releaseCapture);
}
 
```

## 「[static おじさん](http://d.hatena.ne.jp/ryoasai/20110702/1309600182)」ですみません．ただ，これだと texture に書き込んでいる最中（ロックされているとき）に texture を読み出そうとするとブロックしてしまうので，その場合は texture を読まずに以前に読み込んだテクスチャを使うようにしたいと思いました．


<pre>  // テクスチャ取得
if (pthread_mutex_trylock(&amp;mutex) == 0) {
glTexSubImage2D(GL_TEXTURE_2D, 0, 0, 0, width, height, format, GL_UNSIGNED_BYTE, texture);
pthread_mutex_unlock(&amp;mutex);
}
```


## ですが pthread_mutex_trylock() がずーっと EBUSY を返してたまにしか 0 を返さないので，テクスチャが全然更新されません．カメラ側のスレッドでもロックされているみたいで，フレームを取りこぼすこともあります．そもそも OpenCV がスレッドセーフかどうかも知らないし（今回はカメラ１台だから大丈夫だと思ったんだけど）．謎が多い．教えて偉い人．
