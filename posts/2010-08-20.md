```
## すると「プロジェクト名Dlg.cpp」ファイル (ここでは GLsampleDlg.cpp) にメンバ関数 (ここでは SetupPixelFormat()) が追加され, ソースコードを編集する状態になります. ここで SetupPixelFormat() の内容を実装します.


<pre>BOOL CGLsampleDlg::SetupPixelFormat(HDC hdc)
{
<em>PIXELFORMATDESCRIPTOR pfd = {</em>
<em>sizeof(PIXELFORMATDESCRIPTOR),  // PFD のサイズ</em>
<em>1,                              // バージョン</em>
<em>PFD_DRAW_TO_WINDOW |            // ウィンドウに描画する</em>
<em>PFD_SUPPORT_OPENGL |            // OpenGL を使う</em>
<em>PFD_DOUBLEBUFFER,               // ダブルバッファリングする</em>
<em>PFD_TYPE_RGBA,                  // RGBA モード</em>
<em>24,                             // カラーバッファは 24 ビット</em>
<em>0, 0, 0, 0, 0, 0,               //  (各チャンネルのビット数は指定しない) </em>
<em>0, 0,                           // アルファバッファは使わない</em>
<em>0, 0, 0, 0, 0,                  // アキュムレーションバッファは使わない</em>
<em>32,                             // デプスバッファは 32 ビット</em>
<em>0,                              // ステンシルバッファは使わない</em>
<em>0,                              // 補助バッファは使わない</em>
<em>PFD_MAIN_PLANE,                 // メインレイヤー</em>
<em>0,                              //  (予約) </em>
<em>0, 0, 0                         // レイヤーマスクは無視する</em>
<em>};</em>
  
<em>int pf = ChoosePixelFormat(hdc, &amp;pfd);</em>
  
<em>if (pf != 0) return SetPixelFormat(hdc, pf, &amp;pfd);</em>
  
<em>return FALSE;</em>
```


## この関数では, コンピュータが備える OpenGL のサブシステムが用意しているピクセルフォーマットをの中から, 変数 pfd に設定した仕様を満たすものを ChoosePixelFormat() を使って選択し, 見つかったものの番号 pf を SetPixelFormat() で現在のデバイスコンテキスト hdc に設定します. 見つからなければ, pf は 0 になります.


<div class="figure"><img src="mfc/mfc25.gif" alt="関数 SetupPixelFormat() の実装" width="640" height="400"></div>

## しかし, 繰り返しになりますが, このプログラムでは WM_SIZE イベントが OnInitDialog() 実行後に呼び出されることはありません. もし, ウィンドウのサイズを変更できるようにした場合は, OnSize() の引数 cx, cy を使って MoveWindow() などにより OpenGL の表示を行っている Picture Control のサイズを変更することになります. その場合は, 変更するサイズを使って width や height を求めればよいので, わざわざ GetClientRect() を使う必要はありません.

でも, ウィンドウのサイズを変更できるようにするには, すべてのコントロールの配置を計算し直さないといけないんですかね. もう考えるのはめんどくさいので, これは考えないことにします．
