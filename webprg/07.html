１．Web アプリケーション

１．１　Web アプリケーションとは

通常の Web ページは内容が固定しているのに対して，Web アプリケーションでは Web クライアントが Web サーバにアクセスした時点で，Web サーバが Web ページの内容を動的に生成します．これによって，ユーザによるデータ入力や，そのときの状況に応じて，Web ページの内容を変化させることができます．

このように動的な Web ページを生成する場合を Web アプリケーションといい，Web アプリケーション作成のためのプログラミングを Web プログラミングといいます．ここで HTML は，単に Web ページのレイアウトを行うだけでなく，Web アプリケーションのユーザインタフェースの記述にも使用されます．

Web アプリケーションの作成には，機能を Web サーバ側で実現するものとして CGI (Common Gateway Interface)，SSI (Server Side Include)，PHP (PHP:Hypertext Preprocessor)，mod_perl，Java Servlet など，Web クライアント側で実現するものとして JavaScript，Java Applet など，様々なものが利用できます．この講義では，これらのうち最も基本的な CGI と，手軽に利用できる JavaScript について解説します．

１．２　商品発注システムのイメージ

この講義では，前回作成した架空のお店の Web ページに，商品発注の機能を追加します．そこで最初に，お店のトップページ (shop.html) に，「商品選択ページ」へのリンクを張った，「商品の注文」というアンカー（他の Web ページへのリンクを張った文字列や画像など）を追加します．

  • 「商品選択ページ」では，商品在庫情報のデータベースから取り出したデータをもとに，商品リストを表示します．ここでデータベースからデータを取り出して，商品リストの表示や発注データの入力を行うための Web ページを生成するプログラムを呼び出します．
  • 「商品選択ページ」に発注データを入力した後，そのページにある「確定」のボタンを押すと，入力したデータを Web サーバに送り，そのデータを受け取るプログラムを呼び出します．このプログラムは送られたデータを受け取った後，その内容をもとに発注内容の確認と送付先情報の入力を行う「注文確認ページ」の Web ページを生成します．
  • 「注文確認ページ」に送付先データを入力した後，そのページにある「確定」のボタンを押すと，入力したデータを Web サーバに送り，商品在庫情報のデータベースを更新した後，「受注メッセージ」の Web ページを生成します．

５．CGI プログラムの作成

ここまでで作成してきた各ページは，商品在庫情報のデータベースを参照したり，ユーザが入力したデータをサーバ側に送ったりしていませんでした．だから，「商品選択ページ」はいつ見ても同じ内容しか表示されませんし，「商品選択ページ」で入力したデータが「注文確認ページ」の内容に反映されることもありません．

そこで，「商品選択ページ」は通常の HTML ファイルではなく，CGI (Common Gateway Interface) と呼ばれるプログラムによって生成するようにします．また「注文確認ページ」も，「商品選択ページ」で入力されたデータをもとに，CGI によって生成するようにします．

ただし，実際の CGI プログラムは明日以降の講義で作成します．ここでは，仮のプログラムを用意しておきます．
